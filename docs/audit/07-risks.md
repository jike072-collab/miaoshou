# 模块0审计：风险清单

审计时间：2026-06-18

本阶段只做审计与文档整理，不修改业务代码、数据库或页面。

风险等级口径：

- 高：可能阻断首版主流程、导致重复采集/误发布/数据严重不一致或需要平台真实适配。
- 中：会造成失败、误判、用户困惑或需要人工介入，但可通过异常处理继续。
- 低：影响局部体验或维护成本，短期不阻断首版。

## 2. 首版最大的外部平台风险

1. 1688 页面结构变化或加载不完整。
   - 影响搜索和候选提取，是流程前半段核心。

2. 妙手页面结构变化或插件失效。
   - 影响采集箱写入，是流程后半段核心。

3. 登录过期和验证码阻塞。
   - 项目不能绕过验证，只能暂停等待人工，因此必须把等待和恢复做清楚。

## 3. 首版最大的三个数据一致性风险

1. 外部采集成功但本地未记录。
   - 会导致恢复或重试时重复采集。

2. 本地记录成功但妙手未实际采集。
   - 会造成假成功和错误历史去重。

3. 多次运行/多标签页/重试污染同一候选状态。
   - 当前缺统一 run_id 和候选级锁，结果统计也按全库聚合。

## 4. 风险处理优先级

首版必须优先处理：

- 外部平台页面变化和验证阻塞的诊断/暂停/恢复。
- 采集成功回读或至少状态语义降级。
- 重复采集幂等保护。
- 本次运行结果统计隔离。
- 图片质检保守策略。
- 本地 DB/config 备份。

首版可以延后处理：

- 五国定价和汇率运费。
- 发布结果中心。
- 多店铺批次。
- 高级图片接口模板。
- 大规模自动化和并发优化。
