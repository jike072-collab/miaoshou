# 妙手内部上架流程说明

## 全流程状态

系统按同一条状态链推进商品：

候选导入 -> 待补数据 -> 可评分 -> 已评分 -> 可采集 -> 已采集 -> 待生图 -> 待审核 -> 图片通过 -> 可铺货 -> 批次预检 -> 演练通过 -> 真实发布 -> 发布完成 / 失败处理

后端统一使用 `candidate_summary()`、`get_candidate_workflow_status()`、`get_product_workflow_status()`、`image_workbench_summary()`、`batch_preflight()` 和 `publish_results_summary()` 计算状态。前端首页、选品雷达、图片工厂、铺货控制台和失败任务中心应只展示这些接口返回的状态，不单独推断关键门禁。

## 阶段规则

| 阶段 | 进入条件 | 完成条件 | 阻塞条件 | 处理方式 |
| --- | --- | --- | --- | --- |
| 候选导入 | 导入 1688 链接或找品任务写入 candidates | 生成候选记录 | 链接无效、重复或来源不可读 | 重新导入或从来源补全 |
| 待补数据 | 候选缺标题、类目、成本、重量、销量、评分、发货时效、图片或 SKU | 必填数据完整 | 必填字段缺失 | 批量检查、来源补全、人工补数据 |
| 可评分 | 基础数据完整 | 生成五国 evaluations | 目标市场数据不足 | 补目标售价和市场样本 |
| 已评分 | 已有五国评分 | 至少一国达标或明确不达标 | hard block、低置信度、毛利不足 | 采集达标国家、转人工复核或跳过 |
| 可采集 | 至少一国满足分数、置信度和无 hard block | 创建采集任务或正式商品 | 基础数据不完整、重复采集任务 | 批量采集达标国家 |
| 已采集 | 正式商品进入 products | 商品有五国版本 | 采集失败、插件异常 | 失败任务中心重试或转人工 |
| 待生图 | 正式商品无审核图 | 创建生图任务 | 缺主图、图片接口配置错误 | 上传图片或修复生图配置 |
| 待审核 | 有生成图或上传图但未审核通过 | 至少一张 `approved=1` 且非 rejected | 审核驳回、图片不足 | 审核通过、驳回后重生成 |
| 图片通过 | 满足最低审核图片要求 | 可进入批次预检 | 无 | 创建铺货批次 |
| 可铺货 | 商品图片通过且国家版本可用 | 批次预检通过 | 缺图、缺售价、缺库存、缺仓库、店铺不可用、风险拦截 | 修复预检风险 |
| 批次预检 | 批次选择商品和店铺 | `batch_preflight.ready=true` | error 级风险 | 修复阻塞项后重新预检 |
| 演练通过 | dry_run 批次准备并确认 | 批次状态 `completed_dry_run` 或 publish key 为 `dry_run` | 演练失败、页面结构变化、登录失效 | 失败任务中心重试或转人工 |
| 真实发布 | live 批次预检通过且演练通过或明确跳过演练 | publish key 为 `published` | 未通过预检、未处理失败、确认短语错误 | 强确认后执行，失败进入任务中心 |
| 发布完成 / 失败处理 | 自动化或发布结果写入 | 成功结果可追踪，失败已处理 | 未处理失败任务 | 重试、跳过、标记已处理、转人工 |

## 状态一致性约定

- 图片未审核通过时，`batch_preflight()` 必须返回缺图阻塞，前端不能显示为可铺货。
- 自动化失败、生图失败、图片驳回、批次失败和发布 key 失败统一进入 `publish_results_summary().failures`。
- 首页“发布结果与失败处理”的失败数量引用统一失败列表，不只统计 automation_runs。
- 批次创建和真实发布必须以后端预检和确认 gate 为准，前端展示只作辅助。
- 真实发布确认文本固定为 `CONFIRM {商品数}x{店铺数} LIVE`；演练为 `CONFIRM {商品数}x{店铺数} DRY`。

## 测试数据

运行以下命令可生成内部联调样本：

```bash
python3 scripts/seed_workflow_demo.py
```

脚本会写入当前 `WORKBENCH_DATA_DIR`，未设置时写入项目 `data/`。样本包含：

- 缺数据候选
- 未达标候选
- 达标可采集候选
- 已采集正式商品
- 待审核图片和审核通过图片
- 可铺货商品
- 失败批次和失败自动化任务

生成后打开首页和铺货控制台，应能在全流程总览、候选队列、图片工厂、批次预检和失败任务中心看到一致的状态。
