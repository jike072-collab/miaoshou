# 本地运行指南

`miaoshou` 是本地运行的内部选品采集工具。当前真实模式只做到 1688 找品、候选筛选、标题清洗、图片判断，以及把合格商品放入妙手采集箱/待处理区；不会发布商品。

## 一键启动

Windows 环境双击项目根目录的 `run.bat`。

脚本会自动完成：

- 检查 Python。
- 创建或复用 `.venv`。
- 安装 `requirements.txt`。
- 创建 `data/`、日志、截图、图片、专用 Chrome Profile。
- 初始化本地数据库。
- 启动本地服务并打开 `http://127.0.0.1:8000`。

如果是在 macOS 或手动运行，可在项目目录执行：

```bash
python3 scripts/bootstrap.py
python3 app.py
```

默认地址是 `http://127.0.0.1:8765`。Windows 一键脚本固定使用 `http://127.0.0.1:8000`。

## 首次使用

1. 打开首页顶部的“环境状态”。
2. 点击“启动专用Chrome”。
3. 在弹出的 Chrome 里人工登录 1688 和妙手。
4. 如果出现验证码、短信验证或人机验证，只能人工完成。
5. 回到工具页点击“我已完成登录，重新检测”。
6. 确认页面顶部显示 `no_publish=true`、`禁止发布=true`。
7. 点击“开始自动找品采集”。

## 配置

配置文件在 `data/config.json`。首次运行会从 `data/config.example.json` 自动生成。

关键配置：

- `mode`: 默认 `real`。
- `dry_run_collect`: 默认 `true`。
- `collect_to_box_only`: 默认 `true`。
- `no_publish`: 默认 `true`。
- `max_items_per_run`: 每轮最多 10 个商品。
- `max_pages_per_keyword`: 每个关键词最多 2 页。
- `keywords`: 1688 搜索关键词。

模块 10 的真实联调会强制小批量：每轮最多 10 个商品、每个关键词最多 2 页。即使配置写得更大，系统也会收紧到安全上限。

## 运行中怎么看

首页第一屏就是自动化控制台：

- 当前 run_id、关键词、页码、当前商品、当前步骤。
- 总进度和当前商品进度。
- 搜索到、保存候选、重复跳过、风险阻断、标题清洗、图片准备、进入采集箱、失败数量。
- 运行日志和失败处理。

失败商品可以查看详情、打开截图、复制错误信息、重试、标记跳过或标记已处理。

## 常见暂停

- Chrome/CDP 未连接：点击“启动专用Chrome”。
- 1688 未登录：在专用 Chrome 手动登录 1688。
- 妙手未登录：在专用 Chrome 手动登录妙手。
- 验证码/短信/人机验证：人工完成后回到页面重新检测。
- 妙手页面出现“发布/上架/提交/确认发布”等危险按钮：系统会暂停，不点击，并要求校准安全采集箱入口。
